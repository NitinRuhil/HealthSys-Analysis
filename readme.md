# System Analysis Project Repository

This repository contains the main documents for a **System Analysis** course project. The files show the full analysis and design process, from project planning to final presentation.

All documents are provided in PDF format for easy viewing.

---

## Repository Contents

* **Project_Charter.pdf**
  Defines the project purpose, scope, objectives, stakeholders, and timeline.

* **Request_for_Proposal.docx.pdf**
  Describes the problem, background, and expectations for the proposed system.

* **Requirements.docx.pdf**
  Lists the functional and non-functional requirements of the system.

* **UseCases.docx.pdf**
  Explains how users interact with the system through use case scenarios.

* **ProcessModel.docx.pdf**
  Shows system workflows and business processes.

* **Data Model.pdf**
  Presents the database structure, entities, and relationships.

* **UIPrototypes.docx.pdf**
  Displays user interface mockups and screen designs.

* **Website.pdf**
  Contains the website concept or final web design for the system.

* **FinalPresentation.pdf**
  Summarizes the complete project and key findings.

* **CSC375-Slides-g18.pdf**
  Course reference slides used during the project.

---

## How to Read the Project

1. Begin with **Project_Charter.pdf** to understand the project goals.
2. Review **Requirements** and **Use Cases** to learn what the system must do.
3. Study the **Process Model** and **Data Model** for system structure and flow.
4. View **UI Prototypes** to see how users will interact with the system.
5. Open the **Final Presentation** for a complete overview.
